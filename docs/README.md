# 文档总览（工程治理）

本目录用于沉淀 `NovelToST` 项目的工程治理规范，先聚焦四件事：

1. CI/CD 规划
2. 协作规范
3. Ignore 与本地环境策略
4. 仓库历史清理策略

## 文档索引

- [CI-CD规划.md](./CI-CD规划.md)
- [协作规范.md](./协作规范.md)
- [Ignore策略.md](./Ignore策略.md)
- [仓库历史清理.md](./仓库历史清理.md)

## 使用建议

- 新成员先读：`协作规范.md`
- 改工作流前先读：`CI-CD规划.md`
- 提交前出现“哪些文件该不该进仓库”的疑问时，查：`Ignore策略.md`
- 准备做历史重写时，必须先读：`仓库历史清理.md`

## 维护原则

- 文档优先于口头约定。
- 规则变更需要 PR，并在文档中补充“变更日期 + 原因”。
- 文档内容以“可执行、可检查”为准，避免空泛表述。
