### 0.1 本轮新增设计标记：@ 引用上下文模式（Mention Context Mode）

- ✅ 方案状态：**Outline Chat 已实现，并扩展到“当前启用世界书条目”级引用**。
- 🟡 实际接入范围：`Outline Chat` + `Worldbuilding` 协作输入（生成页仍未接入）。
- ✅ 交互方式：输入 `@` 后弹窗选择（不要求用户手写前缀语法）。
- ✅ 上下文策略：发送时**固化快照**，保证重试/审计可复现。
- ✅ `Worldbuilding` 侧同样注入 `引用上下文（来自@）：` 分区。

设计约束（本轮冻结）：

1. `@` 仅代表“结构化引用”，不是立即内联大文本。
2. 引用对象通过 provider 解析（worldbook/worldbook_entry/setup/storyline/node/detail）。
3. 发送前统一渲染为“引用上下文”区块注入 prompt，并记录到消息调试元数据。
4. 单轮引用数量与单引用长度受限（超限摘要 + warning）。

> 版本归属：该能力规划归入 **v2.2**，见第 8 章 `Phase V2.2-F`。

## 4. AI 解析与命令能力（已增强）

相对初版设计，当前实现已经从“只吃固定字段”升级为“**字段 + 命令总线**”双通道。

### 4.1 解析入口

- ✅ 统一入口：`extractJsonPayload()` + `parseOutlineWorkshopPayload()`。
- ✅ `extractJsonPayload()` 具备容错能力：
  - fenced code block
  - 非 fenced 的平衡 JSON 片段
  - 引号包裹 JSON 字符串
  - 尾逗号/控制字符修复（`escapeUnescapedControlCharsInJsonStrings(...)`
    + `replace(/,\s*([}\]])/g, '$1')`）

### 4.2 结构化更新支持

- ✅ 直接字段：`setup/storylines/nodes(masterOutline)/detailsByChapter`。
- ✅ 命令字段：
  - `setupCommands`
  - `storylineCommands`
  - `nodeCommands`
  - `detailCommands`
  - 通用 `commands`（`type/target/entity + op/action`）
- ✅ 支持 C/U/D 同义写法。
- ✅ 支持 `node.split`（自动扩展为 update/create 序列）。
- ✅ 同轮冲突会产出 `parseWarnings`，并保持可回退语义。

### 4.3 Outline Chat 服务行为

- ✅ 仅允许 `outline_chat` 会话类型。
- ✅ 先记 user message，再请求 AI。
- ✅ 解析成功才入快照；解析失败仅保留文本消息。
- ✅ `setupPatch` 会即时合并到当前 setup。

## 9. 结论（当前发布口径）

当前实现已经满足 **v2 MVP（Outline-first）** 的核心发布条件：

1. 会话式大纲协作闭环可用（消息→解析→快照→应用）
2. 无细纲可运行（warn_fallback）与 strict_block 阻断都可用
3. 快照默认安全应用，不误伤细纲
4. Storyline + Node 的人工收口能力可用

同时，细纲工坊完整形态已迁移至 **v2.2**；Outline→Worldbook 的主动导出继续保留在后续增强阶段。
